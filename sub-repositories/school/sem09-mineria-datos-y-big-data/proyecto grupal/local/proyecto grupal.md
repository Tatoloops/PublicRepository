# PROYECTO PRÁCTICO GRUPAL
NoSQL, MongoDB y Visualización de Datos para la Toma de Decisiones  
## 1. Contexto
Actualmente las organizaciones generan grandes volúmenes de datos provenientes de
distintas fuentes, tales como sistemas transaccionales, aplicaciones móviles, sensores,
plataformas web y redes sociales. Para gestionar esta información de forma eficiente se
utilizan tecnologías NoSQL, las cuales ofrecen flexibilidad y escalabilidad para almacenar
datos de diferentes estructuras.  
MongoDB es una de las bases de datos NoSQL más utilizadas en la industria debido a su
modelo documental y facilidad para manejar datos semiestructurados. Por otra parte, las
herramientas de Business Intelligence (BI) permiten transformar datos en información útil
mediante indicadores, gráficos y paneles de control que apoyan la toma de decisiones.  
En este proyecto los estudiantes deberán aplicar los conceptos estudiados durante la unidad
mediante el diseño e implementación de una solución básica utilizando MongoDB y una
herramienta de visualización de datos.  

## 2. Objetivo General
Implementar una solución de análisis de datos utilizando MongoDB y herramientas de
visualización, aplicando conceptos de bases de datos NoSQL, operaciones CRUD y Business
Intelligence para apoyar la toma de decisiones.  

## 3. Objetivos Específicos  
• Comprender las características de las bases de datos NoSQL.  
• Aplicar el modelo documental de MongoDB.  
• Implementar operaciones CRUD sobre una colección de datos.  
• Diseñar consultas para obtener información relevante.  
• Construir visualizaciones que permitan analizar datos.  
• Interpretar resultados y generar conclusiones basadas en evidencia.

### Etapa 2: Implementación Práctica
**Dataset**  
El conjunto de datos seleccionado deberá contener:  
    •Al menos 300 registros.  
    •Al menos 5 atributos.  

**Implementación en MongoDB**  
El grupo deberá:  
    •Crear una base de datos.  
    •Crear al menos una colección.  
    •Importar el dataset.  
    •Evidenciar la correcta carga de los datos.  

**Operaciones CRUD**  
Realizar y evidenciar mediante capturas de pantalla:  
    •Inserción de registros.  
    •Consulta de registros.  
    •Actualización de registros.  
    •Eliminación de registros.  

**Consultas**  
Implementar como mínimo:  
    • 2 consultas simples.  
    • 2 consultas con filtros.  
    • 1 consulta ordenada.  
Cada consulta deberá incluir:  
    • Objetivo.  
    • Código utilizado.  
    • Resultado obtenido.  
    • Breve interpretación.  

### Etapa 3: Visualización de Datos
Utilizando Power BI, Tableau o Looker Studio, construir un dashboard que permita analizar la información almacenada.  

**El dashboard deberá incluir:**  
    • Al menos 2 gráficos.  
    • 1 indicador KPI.  
    • 1 filtro o segmentador.  
    • Título y descripción general.  
**Análisis**  
Los estudiantes deberán interpretar los resultados obtenidos y responder:  
    •¿Qué información relevante entrega el dashboard?  
    •¿Qué decisiones podrían tomarse a partir de los datos?  
    •¿Qué patrones o tendencias se observan?  

### Etapa 4: Pipeline de Datos
Diseñar un diagrama simple que represente el flujo de información utilizado en el proyecto.  
Ejemplo:  
`Fuente de Datos → MongoDB → Consultas → Dashboard → Análisis`  
El diagrama puede realizarse en:  
    •Draw.io  
    •Lucidchart  
    •PowerPoint  
    •Canva
